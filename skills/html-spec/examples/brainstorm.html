Brainstorm — Onboarding redesign

Six directions for the first-run experience. Pick one and we'll spec it out.

1. Single-screen sign-up

Everything on one page. No multi-step wizard.

Why: kills the drop-off cliff between steps 2 and 3. Risk: form looks heavy at first glance.

2. Chat-first setup

Conversational onboarding. Claude asks, user answers.

Why: users describe their context naturally instead of mapping it to fields. Risk: slower than a form for users who know what they want.

3. Template picker

User picks a workspace template; everything else is preset.

Why: demo-ready in 10 seconds. Best for skeptical evaluators. Risk: templates rot if the product changes.

4. Progressive disclosure

Show the empty app immediately. Onboarding lives in a thin top banner.

Why: users see the value before doing the work. Risk: some users skip onboarding entirely and bounce later.

5. Video walkthrough modal

90-second Loom plays first. Then deposit user into the app.

Why: highest-quality explanation per second. Risk: half of users skip videos. Need a strong fallback.

6. Import-first

Skip the setup. Paste an existing artifact (resume, job link) and we infer everything.

Why: zero typing. Strongest "magic" moment. Risk: inference errors mean the user has to clean up data they didn't create.
